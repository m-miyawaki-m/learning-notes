# 依存性注入 - 学習ログ

## 2024-12-07

### 学習内容
- 依存性注入の概念理解
- 密結合と疎結合の違い
- コンストラクタ注入、セッター注入、フィールド注入の比較

### 理解したこと
- DIは単なる「外から渡す」だけでなく、制御の反転（IoC）という概念が重要
- コンストラクタ注入が推奨される理由は「不変性」と「必須依存の明示化」
- フィールド注入が非推奨なのは、テスト時にSpringコンテナなしでインスタンス化できないため

### つまずいたポイント
- IoC（制御の反転）とDIの関係がやや曖昧だった
  - 解決方法: IoC は概念、DI はその実現手段の一つと理解した

### 実践したこと
- 既存プロジェクトのコードを見直し、フィールド注入している箇所をピックアップ
- 簡単なサンプルコードでコンストラクタ注入に書き換え

### 気づき・メモ
- 実プロジェクトでは意外とフィールド注入が多い
- Lombokの`@RequiredArgsConstructor`を使えばコンストラクタ注入がかなり楽
- 循環依存している箇所があるかもしれない（要調査）
