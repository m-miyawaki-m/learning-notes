# SOLID原則 学習計画

## 学習目標
- SOLID原則の5つすべてを理解し、説明できるようになる
- 実コードでSOLID違反を見抜けるようになる
- リファクタリング時にSOLID原則を適用できるようになる

## 前提知識
- Javaの基本文法
- オブジェクト指向の基礎（継承、多態性、カプセル化）
- インターフェースと抽象クラスの理解

## 学習ロードマップ

### Week 1: 単一責任の原則（SRP）
- [ ] SRPとは何か
- [ ] 「責任」の定義
- [ ] SRP違反の例とリファクタリング
- [ ] 実プロジェクトでSRP違反を探す

### Week 2: 開放閉鎖の原則（OCP）
- [ ] OCPとは何か
- [ ] 拡張に開いて、修正に閉じている設計
- [ ] Strategy パターンとの関係
- [ ] 実例でOCP適用

### Week 3: リスコフの置換原則（LSP）
- [ ] LSPとは何か
- [ ] 継承の正しい使い方
- [ ] LSP違反の典型例
- [ ] 契約による設計

### Week 4: インターフェース分離の原則（ISP）
- [ ] ISPとは何か
- [ ] 太ったインターフェースの問題
- [ ] インターフェースの分離方法
- [ ] 実例でISP適用

### Week 5: 依存性逆転の原則（DIP）
- [ ] DIPとは何か
- [ ] 抽象に依存し、具象に依存しない
- [ ] DIパターンとの関係
- [ ] レイヤードアーキテクチャでのDIP

### Week 6: 総合演習
- [ ] 5原則すべてを適用したリファクタリング演習
- [ ] 実プロジェクトのコードレビュー（SOLID観点）
- [ ] チームメンバーへの説明練習
